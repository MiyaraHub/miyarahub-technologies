Compatibility

Works With Your UniFi Controller

Network Deck supports both UniFi OS consoles and standalone UniFi Network controllers. It auto-detects which endpoints your firmware exposes and adapts the UI to match — features you don't have simply don't appear.

🖥️ UniFi OS Consoles

  • UDM Pro SE (verified)
  • UDM Pro
  • UDM / UDR
  • UniFi Cloud Gateway (UCG)
  • UniFi Dream Machine

🧰 Standalone Controllers

  • UniFi Network 10.x (verified)
  • Self-hosted on Linux / Docker
  • Cloud Key Gen2 / Plus
  • UniFi Network Server (Windows)

🔐 Authentication

  • Local admin username + password
  • Two-factor authentication (TOTP)
  • X-API-KEY for programmatic access
  • No UniFi Cloud account required

📱 Mobile Platforms

  • iOS 13 and up
  • Android 5.0 (API 21) and up
  • Phone and tablet layouts
  • Light and dark themes

Firmware tolerance: Network Deck auto-detects the endpoints your controller exposes and adapts to firmware-specific JSON variants (field-name drift, type coercion, multiple shapes for the integration API). Sub-admin accounts may see some endpoints return 401 — that's a controller permissions limit, not an app limitation.

How It Works

Direct to Your Controller. No Cloud Relay.

Network Deck talks directly to your UniFi controller over your local network (or your VPN). Your credentials and network data never touch a MiyaraHub server.

1

Add Your Controller

Enter your controller's hostname or IP, port (default 443 for UniFi OS, 8443 for standalone), and your local admin credentials. Network Deck auto-detects whether you're on UniFi OS or a standalone controller.

2

Authenticate Locally

Sign in with your local admin username and password. Two-factor authentication is supported end-to-end. Prefer API keys? Use X-API-KEY for programmatic access and skip the cookie flow entirely.

3

Manage at a Glance

Watch the live bandwidth chart, kick a misbehaving client, toggle a firewall rule, bounce a PoE port, invite a Teleport guest, or switch to another site — every action round-trips to the controller and updates the live event stream.

Security & Privacy

Local Only. No Account. No Cloud Relay.

Network Deck is designed with a privacy-first, local-only architecture. No MiyaraHub account, no cloud relay, no data collection beyond crash reporting.

🔒

No Cloud Relay

All commands and live streams happen directly between your phone and your controller over your local Wi-Fi or VPN. Nothing is routed through external servers.

🔑

Standard UniFi Auth

Uses the same local admin credentials (or API key) as your UniFi Network web UI. Credentials are stored in iOS Keychain / Android Keystore on device and never transmitted off-device except to your controller.

👤

No MiyaraHub Account

No sign-up, no email, no MiyaraHub account needed. Install, point it at your controller, and you're in.

📡

Minimal Analytics

Firebase Crashlytics for bug reports only. No usage tracking, no ads, no data sales. Network errors are logged as non-fatal to help us improve firmware compatibility without collecting your network layout.

🛡️

Permissions Respected

The app operates within whatever permissions your UniFi admin account has. Sub-admins see only what the controller allows them to see. No privilege escalation, no backdoor.

📶

Network Permission Only

The only device permission Network Deck requires is network access, used exclusively to connect to your UniFi controller on your local network (or via VPN).

Which UniFi controllers are supported? ▼
Network Deck supports both UniFi OS consoles (UDM Pro / Pro SE, UCG, Dream Machine, UDR, Cloud Key Gen2 / Plus) and standalone UniFi Network controllers running v10.x on Linux, Docker, or Windows. The app auto-detects your platform on connection and adapts the UI to the endpoints your firmware actually exposes — features you don't have simply don't appear. Tested live against UDM Pro SE (UniFi OS 5.0.16) and standalone Network 10.2.105.
How does Network Deck communicate with my controller? ▼
Network Deck uses the same local API that the UniFi Network web UI uses — a combination of the classic /rest and /cmd endpoints (with cookie-based session authentication) plus the newer v2 integration API. It also opens a WebSocket to your controller to stream live events. All communication happens directly between your phone and your controller over your local network or VPN.
Do I need a UniFi cloud account? ▼
No. Network Deck uses local authentication only. Sign in with your UniFi local admin username and password (or an API key). If your controller is only reachable via ui.com cloud login today, you'll want to enable a local admin account in Settings → Admins to use Network Deck. The controller never talks to us, and we never talk to Ubiquiti's cloud on your behalf.
Can I manage firewall rules and port forwards? ▼
Yes. Network Deck has a full firewall stack: policies (allow / block / inspect), port forwards with CRUD, traffic rules and routes, firewall groups, firewall zones (v2 policy matrix), and DPI apps / groups. Everything inline-toggles from the list view — no dialog-tap-dialog flow.
Does it support multi-site? ▼
Yes. If your controller manages multiple sites, Network Deck shows a site switcher at the top. Every feature — dashboard, clients, firewall, VPN — is scoped to the currently selected site. Switch with one tap.
How much will Network Deck cost? ▼
Network Deck will be a one-time purchase at $4.99 on both the App Store and Google Play. No subscription, no ads, no in-app purchases. Pricing may change before public launch.
Can I use it remotely (outside my home / office)? ▼
Network Deck connects to whatever hostname or IP you configure. If your controller is reachable via VPN, Tailscale, WireGuard, or a forwarded port, the app will work remotely. We recommend using a VPN or Tailscale rather than exposing your UniFi controller directly to the internet.
Which platforms are supported? ▼
Network Deck runs on both iOS (13 and up) and Android (5.0 / API 21 and up). Phone and tablet layouts are both supported. The app will launch simultaneously on the App Store and Google Play.
